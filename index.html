CI/CD Pipeline Flow

Example workflow (you can match this with GitHub Actions):

1
Code Push / Pull Request
Triggers pipeline when commits are pushed or PR is opened.
2
Lint / Validate
Check HTML formatting, verify Ansible syntax (optional).
3
Build / Package
Bundle static files as an artifact.
4
Deploy
Copy files to server + reload Nginx via Ansible.
5
Smoke Test
Check HTTP 200 and validate content is served correctly.

Deployment Checklist

Quick reminders before deploying:

  • Ensure server is reachable (SSH)
  • Nginx config is valid
  • Correct permissions for /var/www/html
  • Ansible inventory is correct
  • Rollback plan available (previous artifact)
